# 朝の情報読み上げシステム 要件定義

## 概要

毎朝7:30に自動起動し、ニュース取得・要約やカレンダー予定など複数の情報をプラグイン形式で拡張可能な構造で音声読み上げするシステム。

---

## 機能要件

### 1. スケジュール実行
- 毎日 **7:30** に自動実行する
- 実行方法はOSのスケジューラ（Windowsタスクスケジューラ / cron）またはPython内スケジューラ（`schedule` ライブラリ等）を使用

### 2. ニュース取得
- 対象ソース（以下から選定・設定可能）
  - RSSフィード（NHK, 朝日新聞, 日本経済新聞 等）
  - ニュースAPI（NewsAPI.org 等）
  - Webスクレイピング（対象サイトの利用規約を遵守）
- カテゴリ例：国内、国際、経済、テクノロジー、スポーツ（優先度設定可能）
- 1回の実行で取得するニュース件数：**10〜20件**程度

### 3. ニュース要約
- 取得した記事を **LLM（Claude API 等）** を用いて日本語で要約
- 1件あたりの要約長：**2〜4文**（音声で20〜40秒程度）
- 全体の読み上げ時間目安：**5〜10分**
- 見出し＋要約のセットで読み上げテキストを生成

### 4. TimeTree カレンダー連携
- **TimeTree API** を使用して本日の予定を取得
- 取得情報：タイトル・開始時刻・終了時刻・場所・メモ（設定可能）
- 読み上げ形式例：「本日の予定は〇件です。〇時〇分、〇〇。〇時〇分、〇〇。」
- 予定がない場合は「本日の予定はありません」と読み上げる

### 5. 音声合成・再生
- テキストを音声に変換してスピーカーから再生
- 音声合成方式：**VOICEVOX（ローカル）を第一候補**とする
  - VOICEVOX エンジンをローカルで起動し、HTTP API 経由で音声合成
  - **CPU のみで動作**（GPU は使用しない）
  - VOICEVOX が利用不可の場合のフォールバック：gTTS（Google TTS）
- **再生方式：バッチ事前合成**
  - 7:30 に起動してニュース取得・要約・音声合成を順次処理
  - 全音声ファイルの生成完了後に再生を開始（7:35 前後を目安）
  - リアルタイム合成は不要。合成に数分かかっても許容する
  - 生成した音声ファイルは一時ディレクトリに保存し、再生後に削除
- 再生時間：**5〜10分**
- 再生速度・音量の設定可能（任意）

### 6. エラーハンドリング
- ネットワーク接続不可時のフォールバック（前日キャッシュ再生など）
- APIエラー時のリトライ処理
- ログ出力（実行履歴・エラー記録）

## アーキテクチャ要件：プラグイン構造

### 設計方針

新しい情報ソース（天気・株価・TODO等）を **コアロジックを変更せずに追加できる** プラグイン式アーキテクチャとする。

### プラグインの仕様

各プラグインは共通インターフェース（抽象基底クラス）を実装する。

```python
class BasePlugin(ABC):
    name: str          # プラグイン識別名
    order: int         # 読み上げ順序（小さい順に実行）
    enabled: bool      # 有効/無効（config.yaml で切替可能）

    @abstractmethod
    def fetch(self) -> list[ContentItem]:
        """情報を取得して ContentItem のリストを返す"""
        ...

    @abstractmethod
    def format(self, items: list[ContentItem]) -> str:
        """読み上げ用テキストに整形して返す"""
        ...
```

### 実装済みプラグイン（初期）

| プラグイン名 | 概要 | 順序（デフォルト） |
|---|---|---|
| `NewsPlugin` | RSSまたはAPIでニュース取得・要約 | 2 |
| `TimeTreePlugin` | TimeTree APIで本日の予定取得 | 1 |

### 将来追加が想定されるプラグイン例

- `WeatherPlugin`：天気予報の読み上げ
- `StockPlugin`：株価・為替の読み上げ
- `TaskPlugin`：TodoistやNotionのタスク読み上げ
- `BirthdayPlugin`：誕生日や記念日の通知

### プラグインの有効化設定（config.yaml イメージ）

```yaml
plugins:
  - name: timetree
    enabled: true
    order: 1
    access_token: "YOUR_TOKEN"
    calendar_id: "YOUR_CALENDAR_ID"

  - name: news
    enabled: true
    order: 2
    sources:
      - type: rss
        url: "https://www3.nhk.or.jp/rss/news/cat0.xml"
    max_items: 15
```

## 非機能要件

| 項目 | 内容 |
|------|------|
| 動作OS | Windows 11（主要ターゲット） |
| 言語 | Python 3.10 以上 |
| パッケージ管理 | **uv** を使用（仮想環境・依存関係の管理） |
| 起動方法 | `uv run python main.py` / スケジューラ登録 |
| 設定方法 | 機能設定は `config.yaml`、APIキー等の機密情報は **環境変数** で管理 |
| APIキー管理 | OS の環境変数に直接設定。`.env` ファイルは開発時補助として利用可 |
| 依存ライブラリ | `pyproject.toml` で管理（uv） |

## 未決定事項（要確認）

1. **ニュースソース**：RSS / NewsAPI / スクレイピング のどれを優先するか
2. **LLM**：Claude API を使うか、他のモデル（OpenAI等）を使うか、ローカルLLMにするか
3. **TTS エンジン**：VOICEVOX（ローカル・CPU動作）に決定。話者（キャラクター）の選定が必要
4. **スケジューラ**：Python内（`schedule`/`APScheduler`）か、OS（タスクスケジューラ）か
5. **ニュースカテゴリ**：何を優先するか（経済・テクノロジー重視など）
6. **オフライン対応**：不要なら省略可
7. **TimeTree アクセストークン**：個人用トークンを使用するか（TimeTree APIはOAuth2対応）

## システム構成イメージ

```
[スケジューラ]
     ↓ 7:30 起動
[プラグインランナー]
     ├─ [TimeTreePlugin]  ← TimeTree API → 本日の予定取得
     ├─ [NewsPlugin]      ← RSS / API → ニュース取得 → LLM 要約
     └─ [将来のPlugin...]
     ↓
[原稿生成]               ← 各プラグインの出力テキストを結合・整形
     ↓
[TTS モジュール]          ← 音声合成
     ↓
[再生モジュール]          ← スピーカー出力（5〜10分）
```
